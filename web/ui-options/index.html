Can AI actually understand legacy code?

LegacyCodeBench is the benchmark designed to evaluate how well AI systems can understand and document decades-old legacy software, especially COBOL.

View Rankings Run Locally Read Paper

The real problem: Modernization fails not because conversion is hard, but because the business rules, the intent, the quiet rules are buried inside the legacy code.

You can't modernize what you don't understand. Yet no one measures whether AI actually understands these systems. That's why LegacyCodeBench tests whether AI can accurately extract and explain that knowledge.

Scoring Methodology

Four criteria, weighted by importance. Total score is 0–100.

35%

Behavioral Fidelity

Code generated from the documentation must produce identical outputs to the original COBOL program. This verifies the AI actually understood the logic, not just the syntax.

30%

Structural Completeness

All variables, control flow paths, and dependencies must be documented. We automatically extract these from the source and check coverage.

25%

Semantic Quality

A calibrated language model evaluates clarity, accuracy, and whether a developer could understand the code without reading the original.

10%

Traceability

References to specific lines, variables, or sections must point to real code elements. Fabricated references are penalized.

Run the Benchmark

Evaluate your model locally in four steps.

1
Install the benchmark tool
pip install legacycodebench
2
Download the 200 test programs
legacycodebench download
3
Generate documentation with your model
legacycodebench generate --model your-model
4
Get your scores
legacycodebench evaluate
To submit results to the leaderboard, open a pull request on GitHub with your evaluation output.

Questions

Why do all models struggle with enterprise-level tasks?
Enterprise tasks include real production code with non-linear control flow (GO TO statements), dead code sections, cryptic variable names, and business logic accumulated over decades. Even expert COBOL programmers find these challenging to understand.
How is this different from code generation benchmarks?
Most benchmarks test if AI can write new code. We test if AI can understand existing code—specifically messy, critical legacy systems. We verify understanding through execution: if the documentation is accurate, code generated from it must work identically.
Is the benchmark open source?
Yes. The dataset, evaluation code, and all tools are freely available under an open license. We encourage researchers and organizations to use it for improving AI understanding of legacy systems.
